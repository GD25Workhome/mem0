## 1. 理论基础：什么是“路径查询思维”

关系型思维常是“按表查记录”；图思维更强调“沿关系走链路”。

你可以把路径查询理解成三件事：
1. **起点是谁**（source node）  
2. **允许走哪些边**（关系类型、方向）  
3. **最多/最少走几跳**（hop constraints）

示例：
- `(a)-[:KNOWS*1..3]->(b)` 表示从 `a` 出发，沿 `KNOWS` 关系走 1~3 跳到 `b`

---

## 2. 两个核心语法先记住

### 2.1 可变长度路径

```cypher
MATCH p = (a:User {uid:'u1'})-[:KNOWS*1..3]->(b:User)
RETURN p
LIMIT 20;
```

含义：
- `*1..3` = 跳数范围 1 到 3  
- 会返回符合条件的多条路径（可能重复经过同一终点，但路径不同）

### 2.2 最短路径

```cypher
MATCH p = shortestPath((a:User {uid:'u1'})-[:KNOWS*..10]-(b:User {uid:'u9'}))
RETURN p;
```

含义：
- 在最多 10 跳范围内，找 `u1` 到 `u9` 的最短路径（按边数）
- `-[:KNOWS]-` 是无向匹配，表示允许双向探索

## 4. 练习一：1~3 跳好友推荐（重点）

这是你总纲里的第一个练习目标。

### 4.1 第一步：取 `u1` 的 1~3 跳候选人

```cypher
MATCH p = (a:PathDemoUser {session:'path_practice_260331', uid:'u1'})-[:KNOWS*1..3]->(b:PathDemoUser {session:'path_practice_260331'})
WHERE a <> b
RETURN b.uid AS candidate, length(p) AS hops, p
ORDER BY hops, candidate;
```

观察点：
- `length(p)` 是路径长度（边数）
- 同一个 candidate 可能出现多条路径（不同中间人）

### 4.2 第二步：排除 1 跳直连好友，只保留 2~3 跳推荐

```cypher
MATCH p = (a:PathDemoUser {session:'path_practice_260331', uid:'u1'})-[:KNOWS*2..3]->(b:PathDemoUser {session:'path_practice_260331'})
WHERE NOT (a)-[:KNOWS]->(b)
RETURN DISTINCT b.uid AS recommended, min(length(p)) AS min_hops
ORDER BY min_hops, recommended;
```

这就是一个最小“好友推荐候选池”。

### 4.3 第三步（可选）：按“共同中间节点数量”做简单排序

```cypher
MATCH p = (a:PathDemoUser {session:'path_practice_260331', uid:'u1'})-[:KNOWS*2..3]->(b:PathDemoUser {session:'path_practice_260331'})
WHERE NOT (a)-[:KNOWS]->(b)
WITH b, collect(p) AS paths
RETURN b.uid AS recommended,
       size(paths) AS path_count
ORDER BY path_count DESC, recommended;
```

## 5. 练习二：两用户最短关系链（重点）

### 5.1 基础版：找 `u1` 到 `u9` 的最短链路

```cypher
MATCH p = shortestPath(
  (:PathDemoUser {session:'path_practice_260331', uid:'u1'})
  -[:KNOWS*..10]-
  (:PathDemoUser {session:'path_practice_260331', uid:'u9'})
)
RETURN p, length(p) AS hops;
```

### 5.2 输出为“可读链路字符串”

```cypher
MATCH p = shortestPath(
  (:PathDemoUser {session:'path_practice_260331', uid:'u1'})
  -[:KNOWS*..10]-
  (:PathDemoUser {session:'path_practice_260331', uid:'u9'})
)
RETURN [n IN nodes(p) | n.uid] AS node_chain, length(p) AS hops;
```

### 5.3 找不到路径时怎么办

如果返回空，通常是：
1. 两点不连通  
2. 跳数上限太小（`*..10` 可适当增大）  
3. 关系类型写错（不是 `KNOWS`）

## 7. 常见坑与修正建议

### 坑 1：路径爆炸（返回太多）
- 原因：跳数上限过大、图过密
- 处理：限制跳数、增加起点筛选、先小图验证

### 坑 2：方向写错
- 如果关系是有向，你写成 `->` 可能漏结果  
- 先用无向 `-[:KNOWS]-` 验证连通，再决定是否改回有向

### 坑 3：把推荐和最短路径混为一谈
- 推荐通常是“候选集合 + 打分排序”
- 最短路径只是一个“连通性/最短边数”问题

## 8. 相关扩展知识（建议后续学习）

1. **`allShortestPaths`**：当你需要所有最短解，而不是任意一个  
2. **路径约束**：限制中间节点属性（如只允许同城用户）  
3. **图数据科学（GDS）**：Node Similarity、PageRank、Community Detection  
4. **性能优化**：路径查询前先缩小候选子图，避免全图漫游
