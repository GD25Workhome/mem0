# Neo4j `WITH` 语法实验教程：从造数据到每种典型用法

本文在**你自己的 Neo4j 库**里构造一组**可识别、可一键清理**的演示数据，然后逐条练习 `WITH` 的常见用法。  
建议先在 Neo4j Browser 中执行 `:use neo4j`（或你实际存放业务数据的库名），再按顺序执行。

---

## 0. 为什么用独立标签与 `session` 字段

- 演示节点使用标签 `:WithDemoPerson`，并带属性 `session: 'with_practice_260331'`，避免与 mem0 的 `__Entity__` 等业务数据混在一起。
- 实验结束后可用文末「清理脚本」整批删除，不影响其他图数据。

---

## 1. 造数据（一次性执行）

下面 Cypher 会创建 4 个人、若干 `KNOWS` / `WORKS_AT` 关系，并带数值属性便于后面做计算、聚合、排序。

```cypher
// 1) 先删掉本次练习的旧数据（若第一次跑可跳过，不报错也行）
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
DETACH DELETE n;

// 2) 创建节点
CREATE
  (alice:WithDemoPerson {session: 'with_practice_260331', name: 'alice', score: 10, city: 'beijing', role: 'dev'}),
  (bob:WithDemoPerson   {session: 'with_practice_260331', name: 'bob',   score: 30, city: 'beijing', role: 'dev'}),
  (carol:WithDemoPerson {session: 'with_practice_260331', name: 'carol', score: 25, city: 'shanghai', role: 'pm'}),
  (dave:WithDemoPerson  {session: 'with_practice_260331', name: 'dave',  score: 5,  city: 'shanghai', role: 'dev'});

// 3) 创建关系（带关系属性 weight，便于演示）
MATCH (a:WithDemoPerson {session: 'with_practice_260331', name: 'alice'}),
      (b:WithDemoPerson {session: 'with_practice_260331', name: 'bob'})
CREATE (a)-[:KNOWS {weight: 0.9, since: 2020}]->(b);

MATCH (b:WithDemoPerson {session: 'with_practice_260331', name: 'bob'}),
      (c:WithDemoPerson {session: 'with_practice_260331', name: 'carol'})
CREATE (b)-[:KNOWS {weight: 0.6, since: 2021}]->(c);

MATCH (a:WithDemoPerson {session: 'with_practice_260331', name: 'alice'}),
      (d:WithDemoPerson {session: 'with_practice_260331', name: 'dave'})
CREATE (a)-[:MANAGES {level: 1}]->(d);

MATCH (c:WithDemoPerson {session: 'with_practice_260331', name: 'carol'}),
      (d:WithDemoPerson {session: 'with_practice_260331', name: 'dave'})
CREATE (c)-[:WORKS_AT {team: 'alpha'}]->(d);
```

**验证数据是否写入成功：**

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
OPTIONAL MATCH (n)-[r]-(m)
RETURN n.name, type(r), m.name
ORDER BY n.name;
```

## 2. `WITH` 用法一：投影 + 作用域切割（只把需要的变量传给后面）

**含义**：`WITH` 列出「后面还能用的变量」；没列出的变量在后续子句中不可见。

**示例**：只保留 `name` 和 `city`，不再携带完整节点 `n`。

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WITH n.name AS name, n.city AS city
RETURN name, city
ORDER BY name;
```

**对比**：若你在 `WITH` 后还想用 `n`，必须把 `n` 写进 `WITH`：

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WITH n, n.score AS s
WHERE s >= 20
RETURN n.name, s;
```

---

## 3. `WITH` 用法二：计算列（表达式 + `AS` 别名）+ 紧跟 `WHERE` 过滤

**含义**：先算出临时列，再按该列过滤——与 `mem0` 里 `similarity` 的写法同类。

**示例**：把 `score` 翻倍得到 `double_score`，只保留翻倍后大于 40 的人。

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WITH n, n.score * 2 AS double_score
WHERE double_score > 40
RETURN n.name, n.score, double_score
ORDER BY double_score DESC;
```

---

## 4. `WITH` 用法三：聚合（`GROUP BY` 隐式发生）+ 聚合后再过滤

**含义**：`WITH` 里若出现聚合函数（如 `count`、`avg`），未聚合的列会自动成为分组键。

**示例**：按 `city` 统计人数，只保留人数 ≥ 2 的城市。

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WITH n.city AS city, count(n) AS cnt
WHERE cnt >= 2
RETURN city, cnt
ORDER BY cnt DESC;
```

**示例**：按 `role` 求平均 `score`。

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WITH n.role AS role, avg(n.score) AS avg_score
RETURN role, avg_score
ORDER BY avg_score DESC;
```

---

## 5. `WITH` 用法四：`DISTINCT` 去重

**含义**：对当前行集合去重后再进入下一步。

**示例**：列出本次练习数据里出现过的所有关系类型（无向遍历 `[r]`）。

```cypher
MATCH (:WithDemoPerson {session: 'with_practice_260331'})-[r]-()
WITH DISTINCT type(r) AS rel_type
RETURN rel_type
ORDER BY rel_type;
```

---

## 6. `WITH` 用法五：`ORDER BY` + `LIMIT`（取 TopK / 分页的前一步）

**含义**：常用于「先排序再截断」，只把前几行交给后续子句。

**示例**：分数最高的 2 个人。

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WITH n
ORDER BY n.score DESC
LIMIT 2
RETURN n.name, n.score;
```

---

## 7. `WITH` 用法六：多段流水线（先匹配关系，再展开、再统计）

**含义**：复杂查询拆成多段，每段用 `WITH` 衔接。

**示例**：找出每个「起点人」出发的 `KNOWS` 关系条数。

```cypher
MATCH (a:WithDemoPerson {session: 'with_practice_260331'})-[r:KNOWS]->(b)
WITH a, count(r) AS knows_out
RETURN a.name, knows_out
ORDER BY knows_out DESC;
```

---

## 8. `WITH` 用法七：配合 `CALL { ... }` 子查询（Neo4j 4.1+ 常见）

**含义**：外层 `WITH` 把变量传入子查询；子查询内再 `MATCH` 展开。

下面示例：对每个高分用户（`score >= 20`），收集其一度邻居的名字列表。

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WHERE n.score >= 20
WITH n
CALL {
  WITH n
  MATCH (n)--(m:WithDemoPerson {session: 'with_practice_260331'})
  WHERE m <> n
  RETURN collect(DISTINCT m.name) AS neighbors
}
RETURN n.name, n.score, neighbors
ORDER BY n.score DESC;
```

若你的 Neo4j 版本较旧不支持 `CALL {}` 内联子查询，可改用两次 `MATCH` + `OPTIONAL MATCH` 的写法，或只练习前面 1～6 节。

## 9. 与 `mem0` 里相似的片段（可选对照）

下面**不依赖**你库里的向量，只演示「`WITH` 计算一列再 `WHERE`」的形态，便于和 `graph_memory.py` 里 `similarity` 对照理解：

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
WHERE n.score IS NOT NULL
WITH n,
     toFloat(n.score) / 100.0 AS normalized
WHERE normalized >= 0.2
RETURN n.name, n.score, normalized
ORDER BY normalized DESC;
```

若你已在库中启用 `vector.similarity.cosine` 且节点有 `embedding`，可把 `normalized` 换成真实的余弦相似度表达式。

## 10. 清理演示数据（实验结束后执行）

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
DETACH DELETE n;
```

执行后可再确认：

```cypher
MATCH (n:WithDemoPerson {session: 'with_practice_260331'})
RETURN count(n) AS remaining;
```

期望 `remaining` 为 `0`。

## 11. 小结表

| 用法 | 典型场景 | 本教程对应节 |
|------|----------|----------------|
| 投影 / 作用域 | 只把部分变量或属性传给下一步 | §2 |
| 计算列 + 过滤 | 先 `AS` 再 `WHERE` | §3 |
| 聚合 + Having 式过滤 | `count/avg` 后再 `WHERE` | §4 |
| DISTINCT | 去重后再 `RETURN` | §5 |
| ORDER BY + LIMIT | TopK、分页前截断 | §6 |
| 多段流水线 | 关系统计、复杂展开 | §7 |
| CALL {} 子查询 | 对每个起点做子图展开 | §8 |

## 12. 常见问题

**Q：`WITH` 和 `RETURN` 有什么区别？**  
**A**：`RETURN` 结束查询并输出结果；`WITH` 是中间管道，后面还可以继续 `MATCH`、`WHERE`、`CREATE` 等。

**Q：为什么 `WITH n, expr AS x` 后面不能再用没在 `WITH` 里出现的变量？**  
**A**：这是 Cypher 的作用域规则：`WITH` 会切断作用域，只保留显式列出的绑定。
